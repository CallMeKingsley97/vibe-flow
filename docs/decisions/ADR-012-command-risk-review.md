# ADR-012：命令统计与可解释的高危操作审查

- 状态：Accepted
- 日期：2026-07-15

## 背景

Agent 历史中原本只记录“执行终端命令”这一抽象事件，用户无法知道实际运行了什么，也无法快速发现删除文件、丢弃代码、强制推送或修改系统配置等操作。长会话中这些节点还容易被大量普通工具事件淹没。

## 决策

1. Adapter 在命令工具事件中保留结构化 `commands` 数组，一个工具调用可以包含多条命令。
2. 前端同时展示原始命令和归一化命令族；归一化只用于统计，不替换原始证据。
3. 风险等级分为 `none`、`caution`、`high` 和 `critical`。
4. 每条风险必须包含匹配原因，并能够定位到对应原始 Agent 事件。
5. 高危审查独立于普通统计展示；流程节点使用黄色、橙色和红色区分风险等级。
6. 长会话每页最多展示 8 个阶段，使用缩略导航、上一页、下一页、最近阶段和只看高危控制长度。
7. 风险规则是启发式审查，不阻止 Agent 执行，不替代操作系统权限和人工复核。
8. 规则优先追求低误报和可解释性；新增规则必须附带测试。

## 结果

- 用户可以直接看到 Agent 使用过哪些命令以及使用频率。
- 破坏性和外部写入操作不会被普通流程节点淹没。
- 很长的会话不会一次渲染全部阶段卡片。
- 风险提示保留透明边界，避免把关键词匹配误解为完整安全扫描。
