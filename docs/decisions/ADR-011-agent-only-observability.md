# ADR-011：产品收敛为纯 Agent 本地会话观察

- 状态：Accepted
- 日期：2026-07-15

## 背景

网络请求抓取需要目标进程主动使用代理，或由应用修改系统代理。前者无法覆盖独立启动的 Agent，后者会影响其他应用并引入跨平台恢复、权限和 HTTPS 解密问题。这些复杂度偏离了用户真正关心的 Agent 执行过程。

Codex、Claude Code、Gemini CLI 和 Cursor 已经在本机保存了丰富的消息、工具、Skill、MCP、命令、文件和错误记录。围绕这些数据可以提供更稳定、更容易理解的观察体验。

## 决策

1. Vibe Flow 只读取 Agent 自身已经记录或公开输出的数据。
2. 删除运行时网络代理、请求存储、远程服务拓扑和协议流量解析。
3. 进程启动不再注入代理环境变量。
4. 不规划系统代理、MITM、CA 管理或网络流量抓取。
5. 产品首页优先展示 Agent 统计、执行流程和完整事件。
6. 后续分析围绕来源、项目、阶段、Skill、MCP、工具、命令、文件和错误展开。
7. 数据治理只管理 Vibe Flow 的 Agent 会话副本，不修改 Agent 原始文件。
8. 旧数据库通过 migration 7 删除不再使用的表；旧 migration 文件仅为已安装数据库的 checksum 兼容保留。

## 结果

- 应用启动后即可自动发现和更新会话，不依赖代理配置。
- 不修改系统网络环境，不影响浏览器或其他应用。
- 后端、前端和产品文档共享同一条 Agent-only 边界。
- 工程复杂度集中到 adapter 质量、统一事件、统计、性能和发布稳定性。
