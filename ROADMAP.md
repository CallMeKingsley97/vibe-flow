### M7：跨维度统计对比与洞察

> 在会话级统计之上，支持按 Agent、项目（工作目录）和时间范围聚合，帮助用户比较不同 Agent 的使用方式与执行质量。数据边界仍仅限本地 Agent 历史副本。

- [ ] 定义聚合查询协议：维度（source / project / timeRange）、指标（会话数、事件、工具、命令、风险、Token）与排序；
- [ ] 后端按会话元数据与事件汇总实现聚合查询，避免全量拉事件到前端；
- [ ] 前端 Dashboard 增加全局洞察视图（默认最近 7/30 天，可切换时间范围）；
- [ ] Agent 对比：Codex / Claude / Gemini / Cursor 的会话量、工具构成、错误率与高危操作对比；
- [ ] 项目对比：按工作目录聚合 top 项目，支持跳转到该项目相关会话；
- [ ] 时间趋势：按日/周统计会话活跃、错误与高危操作数量；
- [ ] Skill / MCP / 工具成功率与错误趋势（在来源提供结果时）；
- [ ] 聚合结果自动化测试与 10 万级会话元数据性能基线；
- [ ] 文档补充洞察页使用说明与指标口径。

#### 验收条件

- [ ] 用户可不选中单个会话即可看到跨会话聚合洞察；
- [ ] 切换 Agent、项目或时间范围后结果一致且可解释；
- [ ] 聚合逻辑仅依赖统一会话与 AgentEvent，不引入网络抓取；
- [ ] 空数据、单来源、极端时间范围均有合理空态；
- [ ] 相关查询与聚合有自动化回归，性能有可复现基线。

## 5. 持续质量要求

- Rust 业务逻辑不直接依赖 Tauri Window；
- Adapter 不修改 Agent 原始文件；
- 前后端 IPC 有静态类型和运行时校验；
- 长列表使用分页和虚拟渲染；
- 新增聚合逻辑必须有自动化测试；
- 单来源或单文件失败不得中断其他来源；
- lint、typecheck、format、test、Clippy 和构建持续通过；
- 产品方向变化必须同步更新架构、Roadmap 和 ADR。

## 6. 变更摘要

- Codex、Claude、Gemini 和 Cursor adapter 在来源格式提供工具数据时保留实际命令数组；统计页新增常用命令、最近命令记录和高危操作审查；
